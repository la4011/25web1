🎓 자료구조 & 알고리즘 학습

알고리즘 문제 해결(PS)의 기초가 되는 주요 자료구조와 알고리즘에 대해 학습한 내용을 정리합니다.

트리 (Tree)

트리는 이름처럼 나무를 거꾸로 뒤집어 놓은 것 같은 계층적인 구조를 표현하는 자료구조입니다. 파일 시스템이나 조직도처럼 상하 관계가 있는 데이터를 다룰 때 매우 유용합니다.

  • 노드 (Node): 트리를 구성하는 기본 요소로, 데이터를 담고 있습니다.
  • 간선 (Edge): 노드와 노드를 연결하는 선입니다.
  • 루트 (Root): 트리 구조의 최상단에 있는 노드입니다.
  • 부모/자식 노드: 어떤 노드의 바로 위에 연결된 노드가 부모, 바로 아래에 연결된 노드가 자식입니다.

이진 트리 (Binary Tree)

이진 트리는 트리의 한 종류로, 각 노드가 최대 두 개의 자식 노드만을 가질 수 있는 구조입니다. 왼쪽 자식과 오른쪽 자식으로 구분되며, 많은 효율적인 트리 구조의 기반이 됩니다.

  • 모든 노드의 자식 수가 2개 이하인 트리입니다.
  • 데이터 검색, 정렬 등에 활용되는 '이진 탐색 트리'나 '힙'의 기본 구조가 됩니다.
  • 자료가 없는 노드는 'null'로 표현하기도 합니다.

큐 (Queue)

큐는 줄을 서서 기다리는 것과 같은 방식으로 동작하는 자료구조입니다. 가장 먼저 들어온 데이터가 가장 먼저 나가는 '선입선출(FIFO, First-In, First-Out)' 구조를 가집니다.

  • Enqueue: 큐의 맨 뒤에 데이터를 추가하는 작업입니다.
  • Dequeue: 큐의 맨 앞에서 데이터를 꺼내는 작업입니다.
  • 프린터의 인쇄 대기열, 너비 우선 탐색(BFS) 등 순서가 중요한 작업에 사용됩니다.

스택 (Stack)

스택은 접시를 쌓는 것처럼 데이터를 차곡차곡 쌓아 올리는 구조입니다. 가장 마지막에 들어온 데이터가 가장 먼저 나가는 '후입선출(LIFO, Last-In, First-Out)' 구조를 가집니다.

  • Push: 스택의 가장 위에 데이터를 추가하는 작업입니다.
  • Pop: 스택의 가장 위에서 데이터를 꺼내는 작업입니다.
  • 웹 브라우저의 '뒤로 가기' 기능, 함수의 호출 과정(콜 스택), 깊이 우선 탐색(DFS) 등에 활용됩니다.

이진 탐색 (Binary Search)

이진 탐색은 자료구조가 아닌, 정렬된 배열에서 특정 값을 매우 빠르게 찾아내는 탐색 알고리즘입니다. 배열의 중간 값을 확인하고, 찾으려는 값이 더 작으면 왼쪽 절반, 더 크면 오른쪽 절반을 탐색하는 과정을 반복합니다.

  • 반드시 데이터가 정렬되어 있어야 한다는 전제 조건이 있습니다.
  • 탐색 범위를 절반씩 줄여나가기 때문에 데이터가 아무리 많아도 매우 효율적입니다. (시간 복잡도: O(log n))
  • 'Up & Down' 숫자 맞추기 게임과 원리가 비슷합니다.
